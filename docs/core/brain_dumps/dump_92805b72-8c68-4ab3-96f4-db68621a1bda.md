# Merge: SKYROS PGT (KAIROS + Lovable)

O Operador autorizou o *merge* das duas naturezas do projeto. O frontend gerado (React/Vite) será fisicamente engolido pelo ecossistema KAIROS.

## Arquitetura Unificada (Monorepo)

1. **Frontend (`/pgt-ui`)**: Todo o código do `apex-conductor-main` será movido para uma pasta interna chamada `pgt-ui` na raiz do KAIROS.
2. **Backend (`scripts/dashboard.js`)**: Continuará sendo o cérebro / API Server. Adicionaremos a capacidade do `dashboard.js` servir os arquivos estáticos do React compilado (`/pgt-ui/dist`), eliminando a necessidade de rodar dois servidores no dia a dia.
3. **Desenvolvimento (Dev Mode)**: Configuração de proxy no Vite para que ele bata no backend `localhost:3000` nativamente enquanto codamos no React.
4. **Sem Dependências Externas (Lovable Clean-up)**: Como os dados vêm do Obsidian / Roadmap / Status.md (Single Source of Truth do KAIROS), vamos remover a dependência crua do Supabase do React, simplificando o fluxo para leitura direta do Markdown via KAIROS API.

## Plan de Ação (Execution)

### Fase 1: Migração e Setup
- Renomear a pasta `apex-conductor-main` para `pgt-ui`.
- Atualizar `.gitignore` para ignorar `node_modules` do `pgt-ui`.
- Atualizar o backend (`dashboard.js`) para servir estáticos da pasta `dist` do React.

### Fase 2: Lovable Clean-up & Proxy
- Limpar `supabaseClient.ts` (ou mantê-lo mocado/isolado pra não engasgar sem auth).
- Configurar `vite.config.ts` com o proxy para `http://localhost:3000`.

### Fase 3: Data Binding (O Casamento KAIROS + React)
Essa é a transição dos painéis estéticos para dados reais do KAIROS.
- **Header/Dashboard.tsx**: Fazer chamadas `GET /api/status` (Sync Level, Operacional State).
- **CurrentQuestPanel**: Fazer chamada `GET /api/quests` e popular Bosses (P0).
- **Agenda/Quests**: Mapear as Quests P1/P2 do Roadmap.
- **Questlines/Sanctuary**: Chamar `GET /api/anamnesis` listando os dados do cofre Obsidian.

### Fase 4: JARVIS Terminal Visual
- Integrar o componente de input / chat UI do Lovable (se existir) na rota `POST /api/chat` para falar com a LLM no dashboard.
