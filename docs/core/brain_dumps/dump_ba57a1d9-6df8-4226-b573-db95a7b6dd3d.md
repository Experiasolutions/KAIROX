# Phase 3: DEPLOYMENT — WhatsApp Scheduling Flow

## Goal

Build the first **real, working flow**: a WhatsApp webhook server that receives messages from patients, classifies intent, manages appointments via a local calendar, and replies automatically.

## Proposed Changes

### Tier 1: Webhook Server + Local Calendar

#### [NEW] [whatsapp-server.js](file:///c:/Users/Gabriel/Documents/aios-core-main/aios-core/meu-projeto-ai/scripts/whatsapp-server.js)

Express-like HTTP server (port 3001):
- `POST /webhook/whatsapp` — receives Evolution API webhook events
- `GET /webhook/whatsapp` — health check endpoint
- `GET /api/calendar` — view all appointments
- `GET /api/calendar/:date` — view appointments for a specific date
- `POST /api/calendar/book` — manually book an appointment

Message processing pipeline:
1. Receive incoming message from Evolution API
2. Classify intent (greeting, scheduling, cancellation, question, etc.)
3. If scheduling → check available slots → propose 2 options → book on confirmation
4. If cancellation → find appointment → cancel → confirm
5. Send reply back via Evolution API HTTP

#### [NEW] [calendar-store.js](file:///c:/Users/Gabriel/Documents/aios-core-main/aios-core/meu-projeto-ai/scripts/calendar-store.js)

Local JSON-file calendar (stored at `data/calendar.json`):
- Default clinic hours: Mon-Fri 08:00-18:00, Sat 08:00-12:00
- 30-minute slots, configurable buffer time
- API: `getAvailableSlots(date)`, `book(date, time, patient)`, `cancel(appointmentId)`, `reschedule(appointmentId, newDate, newTime)`
- Auto-generates next 7 days of availability
