# Walkthrough: KAIROS SKY — Segundo Cérebro na Nuvem

## O que foi construído

### 🧠 Knowledge Brain — Memória de Elefante

O KAIROS agora tem um **cérebro persistente** que indexa TODOS os 139+ arquivos do sistema:

| Categoria | Arquivos            | O que indexa                                    |
| :-------- | :------------------ | :---------------------------------------------- |
| `rp`      | 12 RPs estratégicos | Zona de Genialidade, OS RPG, Cloud MVP, etc.    |
| `docs`    | 139+ docs           | Engineering Bible, Manifesto, Agent Flows, etc. |
| `scripts` | 23 scripts          | kairos-boot.js, jarvis-core.js, etc.            |
| `agents`  | Workflows           | Todos os agentes e workflows do squad           |
| `clients` | Projetos            | Experia, Hortifruti, Master Pumps               |
| `config`  | Root configs        | README, package.json, settings                  |

Quando tu pergunta algo via `/ask`, o KAIROS **busca no seu próprio cérebro** antes de responder — full-text search em português no Supabase.

### 📦 GitHub Repo

**[github.com/Experiasolutions/kairos-orchestrator](https://github.com/Experiasolutions/kairos-orchestrator)** — 18 arquivos, pronto para Railway.

### 🗄️ Supabase Schema

12 tabelas + `knowledge_brain` + função `search_knowledge()` + triggers + indexes + seed data real.

### 🔑 Estratégia de 15+ API Keys Google

O `key_rotator.py` suporta N keys. Na `.env` do Railway, configurar:

```
GOOGLE_API_KEYS=key1,key2,key3,key4,key5,key6,key7,key8,key9,key10,key11,key12,key13,key14,key15
```

Cada key do Google AI Studio dá ~1500 req/dia. Com 15 keys: **~22.500 requisições/dia grátis**.

1. Abrir [Supabase Dashboard](https://supabase.com/dashboard) → projeto apex-conductor
