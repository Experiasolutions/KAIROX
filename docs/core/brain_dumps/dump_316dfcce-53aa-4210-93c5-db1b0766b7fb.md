# Implementação: Experia Landing Page & Quiz

Esta etapa concretiza as definições estratégicas e os layouts especificados no `PRD-LANDING-PAGE.md` e `DESIGN-SYSTEM.md`. O objetivo é criar ambas as frentes de captação de clientes (Página de Vendas Regional e o Diagnóstico Interativo) para que a "fachada" operacional do negócio esteja pronta para deploy na Vercel hoje.

## User Review Required
> [!IMPORTANT]
> Estamos inicializando dois novos projetos Vite do zero no diretório de clientes (`clients/experia/...`). Esse diretório ficará isolado do resto da monorepo SKYROS/PGT para não misturar os domínios de deploy (que serão estáticos/Vercel).

## Proposed Changes

Vamos utilizar o template de React com TypeScript via Vite para as duas aplicações, a fim de garantir velocidade de compilação, baixo peso de bundle e flexibilidade total com Vanilla CSS, aplicando a paleta *Emerald / Amber*.

### Experia Landing Page (`clients/experia/landing-page`)

Esta será a aplicação principal de vendas, estruturada de forma componentizada para as 10 seções do PRD.

#### [NEW] Vite Project Scaffold
- Executar: `npx -y create-vite@latest clients/experia/landing-page --template react-ts --no-interactive`
- Instalar dependências e ícones (`lucide-react`, `framer-motion` para animações fluidas na UI).

#### [NEW] `src/index.css`
- Implementar as variáveis do **Control Room Edition Design System** (`--bg-root`, `--accent-emerald`, typography, glassmorphism utilities).

#### [NEW] `src/components/ui/*`
- `Button.tsx`: Variações primárias e secundárias com animações hover neon.
- `GlassCard.tsx`: Container com `backdrop-filter` para métricas e depoimentos.
- `Badge.tsx`: Para status e tagueamento de dados (ex: "Operando OK").

#### [NEW] `src/components/sections/*`
1. `Hero.tsx`: Headline agressiva + botão pro Quiz.
2. `ChaosSection.tsx`: O problema dos 55 pontos de falha.
3. `MetricsSection.tsx`: Tabela comparativa "Manual vs Experia".
4. `Executors.tsx`: Apresentação semântica do Agente, Analyst, Worker e Clone.
5. `Timeline.tsx`: Mapeamento → Configuração → Governança em 3 semanas.
6. `BeforeAfter.tsx`: Visão dual do fluxo operacional.
7. `WasteCalculator.tsx`: State simples perguntando "quantos clientes perdidos" gerando um valor formatado.
8. `SocialProof.tsx`: Placeholders focados na persona.
9. `PricingPlans.tsx`: Flexbox cards (Local vs Regional vs Industrial / Consultar).
10. `FinalCTA.tsx`: Bottom bar ou grande container destacando "Não é IA, é governança".

#### [MODIFY] `src/App.tsx`
- Refatorar o `<App />` para compor as 10 seções sequencialmente com transições suaves.
