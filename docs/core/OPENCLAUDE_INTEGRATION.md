# OPENCLAUDE INTEGRATION PLAN

> Este documento serve como guia definitivo para a integração do OpenClaude Proxy com a engine local, pós-formatação (wipe do appdata do Antigravity).

## 1. Contextualização Inicial
Ao iniciar o novo ambiente limpo, você deve chamar o workflow `/context`. Isso forçará a leitura do arquivo `docs/core/KAIROX_COGNITION_CORE.md`, reinjetando e recuperando toda a inteligência e o estado das decisões (God Pool, Task-First Protocol, arquiteturas do Sovereign Engine e Hydra).

## 2. Inicialização KAIROX / OpenClaude
Com o contexto central recuperado:
- Certifique-se de que os MCP servers estejam rodando corretamente de acordo com o KAIROX_COGNITION_CORE e o `SELF_CONTEXT.md`.
- Verifique a execução isolada sem dependências mortas (AIOX / AIOS antigas residuais no arquivo system do Gemini).
- Inicie os workflows pelo Hub central do OpenClaude orientando sua execução via regras estipuladas no God Pool local.

## 3. Proxy Routing no OpenClaude
A rota de delegação de agentes segue os padrões da inteligência recuperada onde as inferências brutas e geração de código serão alavancadas pelas Keys da nuvem e orquestradas localmente.
